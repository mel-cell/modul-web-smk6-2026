## 1. Detail Node.js & Tooling

Kita menggunakan **Node.js 20.x (LTS - Iron)**. Mengapa? Karena versi ini memiliki stabilitas terbaik untuk library yang kita gunakan.

### Panduan Instalasi Node.js (v20)

<Tabs>
  <TabItem label="Windows" icon="seti:windows">
    1. Unduh installer resmi Node.js (v20 LTS) dari [nodejs.org](https://nodejs.org/).
    2. Jalankan installer dan ikuti petunjuk di layar (pastikan opsi penambahan `PATH` dicentang).
    3. Buka PowerShell atau CMD, lalu verifikasi:
       ```bash
       node -v
       # Target: v20.x.x
       ```
  </TabItem>
  
  <TabItem label="Linux" icon="linux">
    Sangat disarankan menggunakan **NVM (Node Version Manager)** di Linux untuk mencegah konflik versi antar project:
    
    ```bash
    curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
    # Restart terminal Anda
    
    nvm install 20
    nvm use 20
    node -v
    ```
  </TabItem>

  <TabItem label="macOS" icon="apple">
    Anda bisa menggunakan Homebrew atau NVM:
    
    ```bash
    # Menggunakan Homebrew
    brew install node@20
    brew link node@20
    
    # Verifikasi
    node -v
    ```
  </TabItem>
</Tabs>

## 2. Mengapa PNPM adalah Harga Mati?

Berbeda dengan npm yang mengunduh ribuan file redundan untuk setiap project, pnpm menggunakan *content-addressable storage*. 

### Manfaat nyata bagi project SMK6:
*   **Kecepatan Build**: Menghemat waktu hingga 50% saat `pnpm build`.
*   **Integritas Lockfile**: Menjamin bahwa library `payload` versi 3.0.0 tidak akan mendadak update ke versi yang merusak tanpa persetujuan kita.

## 3. Docker & Docker Desktop

Pastikan Docker Engine Anda memiliki alokasi RAM minimal **4GB**. PostgreSQL dan Node.js yang berjalan bersamaan cukup memakan memori.

### Instalasi & Pengecekan Docker:

<Tabs>
  <TabItem label="Windows & macOS" icon="laptop">
    1. Unduh **Docker Desktop** dari [situs resminya](https://www.docker.com/products/docker-desktop/).
    2. Alokasikan setidaknya 4GB RAM pada menu Settings > Resources.
    3. Verifikasi instalasi:
       ```bash
       docker compose version
       # Pastikan v2.x.x ke atas
       ```
  </TabItem>
  
  <TabItem label="Linux" icon="linux">
    Gunakan repositori resmi Docker untuk performa *native*:
    ```bash
    # Instalasi via script resmi
    curl -fsSL https://get.docker.com -o get-docker.sh
    sudo sh get-docker.sh
    
    # Pastikan plugin compose terinstal
    sudo apt-get install docker-compose-plugin
    
    docker compose version
    ```
  </TabItem>
</Tabs>

## 4. Konfigurasi Git Global

Pastikan Anda mengatur identitas Git Anda agar kontribusi Anda tercatat dengan benar:
```bash
git config --global user.name "Nama Lengkap Anda"
git config --global user.email "email@anda.com"
```

## 5. Quick Start: Membuat Project Payload Baru

Jika Anda ingin mencoba membuat sistem seperti SMK6 Engine dari nol, Payload menyediakan perintah yang sangat mudah:

```bash
npx create-payload-app@latest
```

*   **Panduan Lengkap**: [Payload Quick Start Guide](https://payloadcms.com/docs/getting-started/installation)
*   **Video Tutorial**: [Payload YouTube Channel](https://www.youtube.com/@payloadcms)

Setelah menjalankan perintah di atas, Anda akan diajak memilih template dan database. Untuk project SMK6, kita menggunakan database **PostgreSQL**.

## 6. Checklist Sebelum Koding

:::caution[BACA INI]
Jika Anda baru pertama kali melakukan `git clone` project ini, lakukan urutan ini tanpa terlewat:
1. `pnpm install`
2. Buat file `.env`
3. `docker compose up -d`
4. `pnpm generate:types`
5. `pnpm seed` (Hanya jika butuh data demo)
